## 通俗类比库（术语翻译器）

用于将专业术语翻译成零基础用户也能秒懂的大白话。

### Python/FastAPI 生态术语（重点）

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| FastAPI | 高性能 Python Web 框架 | 就像餐厅前台，接待客人点单，把请求分给后厨处理 |
| Pydantic | 数据校验管家 | 就像质检员，数据进来前先检查格式对不对、类型合不合 |
| dataclass | 结构化的数据盒子 | 像填表格，每个格子有固定的名字和类型，不用手写模板代码 |
| ABC（抽象基类） | 能力证书 | 规定子类必须实现哪些方法，像驾照要求必须会倒车入库 |
| Protocol（协议类） | 隐形能力证书 | 跟 ABC 类似但不强制继承，只要有这个方法就算合格 |
| threading.RLock | 可重入的门锁 | 同一时刻只允许一个线程进入，但同一个线程可以反复进出 |
| asyncio | 单线程协程调度员 | 一个人同时盯很多锅，哪个锅开了就去处理，不用傻等 |
| uvicorn | ASGI 服务器 | 就像餐厅大门，接收 HTTP 请求并转交给 FastAPI |
| handler / route | 请求处理函数 | 就像服务员，接到点单后去找对应的厨师 |
| middleware | 请求的安检通道 | 每个请求进来前都要过安检（CORS、鉴权、限流等） |
| dependency injection | 外卖模式 | 需要什么别自己做，告诉框架"我要这个"，它送过来 |
| config.yaml | 项目的配置文件 | 就像餐厅的菜单和营业时间表，改了就能改变行为 |
| requirements.txt | 购物清单 | 记录项目依赖了哪些外部包及其版本 |
| Docker / docker-compose | 集装箱 + 船队 | 把应用和环境一起打包，到哪都能跑 |
| SSE（Server-Sent Events） | 服务器单向推送 | 就像收音机广播，服务器一直说，客户端一直听 |
| context manager | 自动关门器 | with 语句块结束时自动清理资源，不用手动关 |
| decorator | 包装纸 | 给函数外面套一层，不改变内容但增加功能 |
| generator / yield | 水龙头 | 一点一点往外吐数据，不用一次全倒出来 |

### AGI / Agent 项目术语（本专项）

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| Agent | 智能助手的大脑 | 接收用户输入，路由到合适的处理模式，返回结果 |
| RAG（检索增强生成） | 先查资料再回答 | 就像开卷考试，先翻书找到相关内容，再根据内容作答 |
| Embedding | 文字向量化 | 把一段文字变成一串数字，语义相近的文字数字也相近 |
| Milvus | 向量数据库 | 专门存"文字的数字指纹"的仓库，支持相似度搜索 |
| Elasticsearch | 全文搜索引擎 | 百度搜索的核心引擎，对文字内容建立倒排索引 |
| Neo4j | 图数据库 | 专门存"谁跟谁有关系"的数据库，关系是核心 |
| RRF（倒数排名融合） | 综合排名 | 三个评委各给一套排名，按倒数打分综合出最终排名 |
| Hybrid Search | 混合搜索 | 向量相似度 + 关键词匹配 + 知识图谱，三重搜索取最优 |
| ShortTerm Memory | 短期记忆 | 当前对话的上下文窗口，滑动式保留最近 N 轮 |
| LongTerm Memory | 长期记忆 | 从历史对话中提取并持久化的重要信息，支持相似度召回 |
| Preference Memory | 偏好记忆 | 用户习惯和偏好的规则集合，自动从对话中提取 |
| Graph Memory | 图增强记忆 | 用 Neo4j 存储记忆片段及其关系，支持关系推理 |
| Sandbox | 代码沙箱 | 隔离执行环境，AI 生成的代码在里面跑，不影响宿主机 |
| ReAct Mode | 思考-行动循环 | Planner 规划 → Executor 执行 → Generator 生成，多步迭代 |
| Tool Registry | 工具注册表 | 记录有哪些工具可用，按名查找，支持动态注册 |
| Schema / Slot | 提示词模板拼图 | 预定义"提示词的占位符"，运行时按优先级填充 |
| Prompt Context | 运行时提示上下文 | 最终喂给 LLM 的完整提示词，由多个 Source 组装而成 |
| Clean Architecture | 分层式别墅 | domain(地基) → infrastructure(水电) → application(生活) → interfaces(大门) |
| Repo Pattern | 储物柜管理员 | 统一管理数据的存取，业务层不关心数据存在哪 |

### 前端术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 组件 | 积木块 | 一个页面由很多积木拼成，每个积木可以单独用 |
| 路由 | 导航地图 | 告诉程序"这个地址去哪个页面" |
| 状态管理 | 全局记事本 | 所有页面都能读写的共享数据 |
| Props | 父母给孩子的礼物 | 父组件传给子组件的数据，只能看不能改 |
| State | 自己的日记本 | 组件自己的数据，可以随时改 |
| 生命周期 | 人的一生 | 组件从出生（挂载）到死亡（卸载）的过程 |
| 虚拟 DOM | 草稿纸 | 先在草稿上画好，再一次性抄到正式纸上 |
| Hooks | 钩子 | 在特定时机"钩"住，执行你想做的事 |
| SSR | 厨房出餐 | 服务器把页面做好端上来，不是让顾客自己拼 |
| CSR | 自助餐 | 浏览器自己组装页面 |

### 后端术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| API | 服务员 | 前端点菜、后端做菜、API 传菜 |
| 中间件 | 安检通道 | 请求进来前先过一道检查 |
| 数据库 | 仓库 | 存放所有数据的地方 |
| ORM | 翻译官 | 把代码语言翻译成数据库语言 |
| 缓存 | 便利贴 | 把常用信息贴在手边，不用每次去仓库找 |
| 队列/MQ | 排队叫号 | 任务太多先排队，一个一个处理 |
| 微服务 | 专科医院 | 每个服务只干一件事，各司其职 |
| 网关 | 大门保安 | 所有请求先过网关，统一检查和分发 |
| 鉴权 | 查身份证 | 确认"你是谁"以及"你能干什么" |
| 事务 | 要么全做要么全不做 | 银行转账：扣钱和加钱必须同时成功 |
| TCC | 三步走事务 | Try(预留)→Confirm(确认)→Cancel(回滚)，分布式场景下保证一致性 |
| 分布式锁 | 公共厕所的门锁 | 同一时间只能一个人用，防止并发冲突 |
| 幂等 | 按电梯按钮 | 按一次和按十次效果一样，不会重复执行 |
| 降级/兜底 | 备用方案 | 主方案挂了，自动切换到备用方案 |
| 限流 | 景区限流 | 人太多就不让进了，保护系统不被压垮 |
| 熔断 | 保险丝 | 下游服务挂了就断开连接，不要一直重试拖垮自己 |

### 架构模式术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 分层架构 | 流水线 | 每层只干自己的事，按顺序传递 |
| 依赖注入 | 外卖模式 | 需要什么别自己做，让别人送过来 |
| 接口/Interface | 插座标准 | 只规定"长什么样"，不管里面怎么实现 |
| 工厂模式 | 自动贩卖机 | 告诉它要什么，它自动给你造一个 |
| 单例模式 | 全校只有一个校长 | 整个程序只有一个实例 |
| 观察者模式 | 订阅公众号 | 有新消息就自动通知所有订阅者 |
| 策略模式 | 导航选路线 | 同一个目的地，可以选不同的路线（策略） |
| 注册模式 | 挂号 | 先注册自己的能力，需要时按名字找到你 |

### 通用编程术语

| 术语 | 通俗翻译 | 类比 |
|------|----------|------|
| 变量 | 盒子 | 用来装数据的容器，可以换内容 |
| 函数 | 机器 | 放进原料，出来产品 |
| 回调函数 | 留个电话 | 事情办完了打给你 |
| Promise | 承诺书 | 承诺未来给你结果（成功或失败） |
| async/await | 排队等号 | 告诉程序"这里要等一下" |
| 继承 | 家族遗传 | 子类自动拥有父类的能力 |
| 多态 | 一招多用 | 同一个方法，不同对象有不同表现 |
| 递归 | 俄罗斯套娃 | 自己调用自己，一层套一层 |
| 并发 | 一心多用 | 同时处理多件事（不一定真的同时，可能是快速切换） |
| 并行 | 多人同时干活 | 真的同时处理多件事（多核CPU） |
